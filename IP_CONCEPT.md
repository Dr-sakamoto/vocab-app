# Etymon ― 世界観バイブル（独自IP）

> 仮ブランド名 **Etymon（エティモン）**／御三家 **SPEC・PORT・VOC**。
> いずれも差し替え容易な仮確定値。最終決定で `lib/branding.js` を更新すれば全体へ反映する。

## 1. ひとことで

**「ポケモン × 語源辞典（OED）× 真名魔法」。**
モンスターは**英単語の語根（語源）が生きた存在 = Etymon**。
単語を覚える行為が、そのまま世界で力を得る行為になる学習RPG。

## 2. なぜ最強にオリジナルで、説得力が高いのか

英単語は実際に **ラテン語・ギリシャ語の語根**から枝分かれして生まれている。

- `PORT`（運ぶ）→ import / export / transport / portable / report
- `SPEC`（見る）→ inspect / spectator / spectacle / respect
- `VOC`（声）→ voice / vocal / vocabulary / invoke

この **「語根 → 派生語」という現実の構造**を、そのまま **「進化前 → 進化後」** に重ねる。
進化が*事実の語源*なので世界に嘘がなく（＝説得力が最大）、覚えるほど世界が開ける。
既存のいいとこ取り：**ポケモンの収集・進化ループ＋本物の語源学＋真名魔法（名を知る者が縛る）**。
この三点の組み合わせを核に据えた монスター収集RPGは前例がない。

## 3. 設定（プレミス）

太古、すべての言葉は一つの種＝**語根**から生まれた。語根は生きており、
**Etymon** と呼ばれる原初の獣である。言語が育つにつれ語根は派生語へ枝分かれし、
Etymon もそれに連れて姿を変えてきた。**単語を正しく思い出す（記述式クイズ）こと**は、
その語に眠る語根を*目覚めさせる*こと。プレイヤーは語根を見抜き縛る者 **= 銘者（Namer）**。

## 5. 命名規則（152種を体系的に生成する）

各ラインは1つの語根に対応。各段階の名は、その語根の**実在派生語**に接尾辞で
「幼／中／王」の段階感を与える。これによりモンスター名自体が語彙学習になる。

- **幼体**: 語根 + `-let` / `-ling`（例: Speculet）
- **中間**: 接頭辞付き派生語ベース（例: Inspex ← in-spect）
- **完成**: 語根 + `-arch`（王）/ `-on` 等（例: Spectarch）

> 移行時は `lib/monster.js` の各ラインで `name`/`nameEn`/`sprite` と系統を差し替える。
> `id`（lineId・図鑑番号）は多数の箇所から参照されるため**維持**する。

## 7. 祖語（伝説/シンボル枠）

全語根の神話的祖先＝**印欧祖語（PIE）**。実在の概念なので説得力が高い。
名は PIE 慣習に倣いアスタリスクを冠する。例：

- **\*deru-**（堅い・木）→ true / trust / endure / tree の祖。**「\*Deru, 不変の祖」**
- **\*bʰeh₂-**（語る・光る）→ voice/fame 系の祖。**「\*Bha, 始声の祖」**
- **\*gʷeih₃-**（生きる）→ vital/bio 系の祖。**「\*Gwei, 生命の祖」**

これらをミュウ/ミュウツー級のレア枠（`battle.tier: "legendary"|"symbol"`）へ対応づける。
