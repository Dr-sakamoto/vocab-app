# IP差し替え計画（独自IPへの全面移行）

> **収益化のクリティカルパス。** 本アプリはポケモンのIP（種族名・スプライト・
> 世界観用語）に全面的に依存しており、**1円でも収益を得た時点で著作権・商標
> 侵害のリスク**が生じる。商用公開の前提として、第三者IP資産をオリジナルへ
> 置き換える必要がある。

## 監査結果（2026-06 時点）

| 区分 | 内容 | 規模 | 該当箇所 |
| --- | --- | --- | --- |
| 種族データ | 和名＋英名でポケモン種族を直書き | **進化79ライン / 152種** | `lib/monster.js` |
| モンスター画像 | PokeAPI の gen-ii スプライトを**外部直リンク** | 152種 × 複数フォーム | `lib/monster.js`（`SPRITE_ROOT`） |
| 対戦相手画像 | Pokémon Showdown のトレーナー画像を直リンク | 多数 | `lib/storyBattles.js` |
| 効果音 | `public/pokemon-evolve.mp3`（名称・出所とも要差替） | 1 | `public/` |
| 世界観用語 | ポケモン / ボール / 御三家 / マスターボール 等 | 〜60箇所 | `lib/*`, `app/*` |
| ブランド名 | metadata の「ポケモン英単語クイズ」 | 1 | `app/layout.tsx` |

> **重要:** スプライトの外部直リンクは、IP侵害に加えて「第三者サーバーへの
> 依存」というサービス安定性リスクでもある（先方が止めれば全画像が消える）。

## 移行のしかた（データ駆動化が鍵）

幸い種族スプライトは図鑑番号 `id` から URL を機械的に生成しており
（`crystalSprite(id)`）、画像の配信元は1箇所（`SPRITE_ROOT`）に集約されている。
したがって**用語・アセット参照を `lib/branding.js` に寄せ、種族テーブルを
オリジナルへ差し替える**という順序で、コード構造を大きく壊さず移行できる。

## フェーズ計画

- [ ] **P1. 中立化（コードのみ・アート不要）**
  - `lib/branding.js`（作成済み）へ用語・アセット参照を集約。
  - `TERM_GLOSSARY` に沿って「ポケモン」等のUI文言・コメントを独自/中立語へ置換。
  - `app/layout.tsx` のタイトルを `BRAND.name` 由来に変更。
- [ ] **P2. オリジナル種族の定義**
  - 152種ぶんのオリジナル名（和名/英名）と進化ラインを確定。
  - `lib/monster.js` の `MONSTER_LINES` をオリジナル名へ差し替え（`id` 体系は維持可）。
- [ ] **P3. オリジナルアート/音源の用意と自前ホスティング**
  - モンスター画像・対戦相手画像・進化音をオリジナル素材で用意し `/public/` 配下へ。
  - `ASSET_SOURCES.*.migrated` を `true` に更新。
  - 外部直リンク（PokeAPI / Showdown）を全廃。
- [ ] **P4. 公開ゲート**
  - `hasUnmigratedAssets()` が `false` になるまで商用公開しない（CI等で機械チェック）。

## 公開可否の機械チェック

`lib/branding.js` の `hasUnmigratedAssets()` が `true` の間は第三者IP資産が
残っている。リリース前チェック（手動/CI）でこれが `false` であることを必須条件にする。
