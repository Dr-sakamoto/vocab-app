## 1. プロジェクト概要

英単語学習RPG「Etymon」のモンスター命名プロジェクト。  
ポケモンIPからオリジナルIPへの全面差し替えが目的。

- **全82ライン・約152種**に和名（＋英名）をつける
- 内部IDはポケモン名を流用（`bulbasaur`, `charmander` 等）しているが、**ポケモン個体のデザイン・役割・特性は完全無視**。スロットはただの器
- **実装は全名称確定後に一括**。命名フェーズ中は `lib/monster.js` を触らない

## 4. 命名ロジック

### ロジック①：体パーツ × 劇的慣用句（主力・最多採用）

**基本形**：体の部位をオノマトペや幼児語で（ミミ・マナコ・マウス・アンヨ）  
**最終形**：その部位が題材の確立した慣用句・複合語（ジゴクミミ・センリガン・ビッグマウス・イダテン）

> 進化前は「ただの耳」、進化後は「地獄耳」。落差が物語になる。

採用例：
- ミミ → **ジゴクミミ**（地獄耳＝何でも聞こえる）
- マナコ → **センリガン**（千里眼＝遠くまで見通す）
- マウス → **ビッグマウス**（大口叩き）
- アンヨ → **イダテン**（韋駄天＝足が速い）
- ハナスケ → **テングバナ**（天狗鼻）
- オテテ → **センジュ**（千手）

### ロジック②：スケール倒置

極小のモチーフが、宇宙・永遠・伝説スケールの最終形に化ける。

採用例：
- ホシノコ → ヤドリボシ → **アマノガワ**（星の子が天の川に）
- ポカン → **ウツロ**（ぽかんとした穴が宇宙的虚無に）

### ロジック③：動物 × 故事成語

動物名の鳴き声・幼名を基本形にして、その動物が主役の故事成語を最終形に。

採用例：
- コン → **トラノイ**（虎の威を借る狐。こんこん鳴く小狐→虎縞をまとった化け狐）

## 5. 良い名前の3条件

1. **基本形は擬動物化できる**  
   「これを生き物として描けるか」がテスト。体パーツ・生き物・オノマトペ・有機的な現象はOK。人間が作った文化的産物（絹布・枕・暦等）はNG。

2. **最終形は確立した日本語の複合語・慣用句**  
   造語・造語感のある語は避ける。辞書や古典・慣用句集に載っている語を使う。

3. **各段階が単体で完成した名前**  
   接尾辞の機械的付加（〜くん/もん/ドン）はしない。  
   並べると進化の物語になること（カタコト→言霊、コン→トラノイ等）。

## 6. NG基準（実際に試みて却下した例）

| 提案 | 理由 |
|------|------|
| シロタエ（白妙=白い絹） | 人工物・布。生き物として描けない |
| ユメマクラ（夢枕） | 人間文化的概念。体がない |
| ユウナギ（夕凪） | 大気現象。形がない |
| ナギ→シジマ | 凪・静寂という抽象。ぴったりの水生生物がなく断念 |
| ダルマ→ヒダルマ | ポケモン既存名と重複（Darumaka/Darmanitan） |
| ウツロ→ナラク | スケールダウン感。宇宙的虚無から仏教地獄への格落ち |
| カミコ→オリヅル | 和紙・鶴は美しいが、モンスターとしての生き物感に欠ける |
| テマリ→テマリウタ | 手毬は人工物。擬動物化が難しい |

## 7. 動物イメージの原則

名前＝**概念**、ビジュアル＝**動物** という二層構造。

- 進化前後で動物は変えない（全段階で一貫）
- 名前に動物が埋め込まれている場合はそのまま採用（オリヅル→ツル、テングバナ→テングザル、ヨトウガ→蛾）
- 選び方：その概念を体現する動物を選ぶ（ミミ=耳→ウサギ、マナコ=眼→メガネザル、ハラ=腹→タヌキ）
- 概念と動物の組み合わせに意外性・必然性があるほど良い

## 8. バランス管理（多角的パラメータ）

名前全体をこれらの軸で俯瞰し、空白を埋める：

| パラメータ | 現状 |
|-----------|------|
| 色彩 | **未着手**（赤・白・青等ゼロ） |
| 硬さ | イシアタマ方向で検討中 |
| 大きさ感 | 小〜中。極小・極大が薄い |
| 生物種 | 人体パーツ偏重。水生・植物・虫が少ない |
| 時代感 | 古典和語中心。現代語・外来語ゼロ |
| 抽象度 | 低〜中。純粋抽象が薄い（ウツロで1つ） |

## 9. 単独キャラのルール

- UMAは**地域が重複しないように**（例：クラーケンはユニコーン=欧州・ニンゲン=海洋と重複で削除）
- 単独キャラが多すぎないよう注意（現在8体が上限感）
- UMAと一般単独は別管理（UMA=Section E的扱い、ニンゲン等は別セクション単独）
